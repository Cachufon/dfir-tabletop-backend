# Cloud Scope Area Data Sources

This document describes the minimum and optional data sources required to run the Cloud scope area effectively.

## Minimum Data Sources (choose any that apply based on customer cloud)

- AWS CloudTrail (or equivalent audit trail)
- Azure Activity logs (or equivalent)
- GCP Admin / audit logs (or equivalent)

## Optional Data Sources

- IAM config snapshots
- Cloud security posture tool exports (if available)
- Kubernetes audit logs (if in scope)
- VPC flow logs / proxy logs (if available)
