# INTERNAL - TTX Scenarios

This folder contains **non-client-specific** tabletop scenarios in YAML.

## Rules
- Do not commit client-tailored scenarios here.
- Do not include real client names, systems, domains, emails, or incidents.
- Keep scenarios generalized and reusable.

## Authoring notes
- Prefer clear, decision-focused injects.
- Each inject should force a decision, reveal an information gap, or validate a process.
- Use `facilitator_notes` for branching guidance and extra context.

## Example scenarios
- `example_ttx_saas_identity_compromise.yaml` (`SaaS / Identity`)
- `example_ttx_ransomware_extortion.yaml` (`Ransomware / Extortion`)
- `example_ttx_insider_data_exfiltration.yaml` (`Insider / Data Exfiltration`)
- `example_ttx_cloud_iaas_compromise.yaml` (`Cloud (IaaS) Compromise`)
- `example_ttx_third_party_supply_chain.yaml` (`Third-Party / Supply Chain`)
- `example_ttx_web_application_api_breach.yaml` (`Web Application / API Breach`)
